# syncfield-swift v0.2 — Plan C: Insta360CameraStream (wrist camera)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a new SPM target `SyncFieldInsta360` that exposes an `Insta360CameraStream` conforming to `SyncFieldStream`. The adapter uses INSCameraSDK (Insta360's binary xcframeworks already used by egonaut) to BLE-trigger remote recording on a Go 3S camera, and downloads the resulting mp4 file over WiFi during the `ingest()` phase with progress + cancellation support. Timestamps of the BLE ACK are recorded so the server can align the wrist-camera mp4 against host-time streams.

**Architecture:** `SyncFieldInsta360` is a separate SPM target (kept separate because it drags in ~150 MB of binary framework dependencies the non-Insta360 user doesn't want). The adapter class wraps two existing egonaut managers — `Insta360CameraManager` (BLE control) and `Insta360WiFiTransferManager` (WiFi download) — porting them into the SDK verbatim with a clean `SyncFieldStream` facade.

**Tech Stack:** INSCameraSDK.xcframework, INSCoreMedia.xcframework, INSCameraServiceSDK.xcframework (all from Insta360's iOS SDK distribution), CoreBluetooth, URLSession for download.

## Pre-requisite decision — xcframework distribution strategy

Before implementation, the repo owner must pick one:

### Option A: Customer-supplied binary (recommended for distribution)

`Package.swift` declares `.binaryTarget(name: "INSCameraSDK", ...)` but the actual binary is **not** committed to the repo. The customer consuming `SyncFieldInsta360` drops the xcframeworks into their app bundle and the Swift module links at their build time. `swift build` in this repo then *cannot* compile the `SyncFieldInsta360` target alone — a separate `#if canImport(INSCameraSDK)` guard inside `Insta360CameraStream` makes the whole type empty when the module is absent, so library consumers who don't need Insta360 aren't blocked. CI builds the target only when a stubbed/local binary is present.

- Pros: SDK repo stays slim, customers own their Insta360 license
- Cons: `swift build` of the full package requires the binary present locally

### Option B: In-repo binary (simpler to develop + CI)

Commit the xcframeworks directly into `syncfield-swift` under `Binaries/Insta360/`. `Package.swift` wires them as `.binaryTarget(name: "INSCameraSDK", path: "Binaries/Insta360/INSCameraSDK.xcframework")`. Repo grows by ~150 MB.

- Pros: `swift build` just works; CI is trivial; unified versioning
- Cons: repo bloat, LFS likely required, Insta360 SDK licence implications

### Option C: Path dependency on egonaut (development-only)

Use a relative path pointing at egonaut's existing copies. Fine for local dev on this machine, brittle for anyone else. Best as a stop-gap while choosing between A and B.

**This plan assumes Option A (customer-supplied).** Implementation includes `#if canImport(INSCameraSDK)` guards so Insta360CameraStream is declared but no-op without the module. If Option B is chosen, the guards are harmless (canImport is always true) — only `Package.swift` changes.

```swift
// Sources/SyncFieldInsta360/Insta360CameraStream.swift
import Foundation
import SyncField

public final class Insta360CameraStream: SyncFieldStream, @unchecked Sendable {
    public nonisolated let streamId: String
    public nonisolated let capabilities = StreamCapabilities(
        requiresIngest: true, producesFile: true,
        supportsPreciseTimestamps: true, providesAudioTrack: true)

    private var healthBus: HealthBus?
    private var bleAckMonotonicNs: UInt64 = 0
    private var cameraFileName: String?

    #if canImport(INSCameraSDK)
    private let ble = Insta360BLEController()
    private let wifi = Insta360WiFiDownloader()
    #endif

    public init(streamId: String) {
        self.streamId = streamId
    }

    public func prepare() async throws {}

    public func connect(context: StreamConnectContext) async throws {
        self.healthBus = context.healthBus
        #if canImport(INSCameraSDK)
        try await ble.pair()
        #endif
        await healthBus?.publish(.streamConnected(streamId: streamId))
    }

    public func startRecording(clock: SessionClock,
                               writerFactory: WriterFactory) async throws {
        #if canImport(INSCameraSDK)
        let (ackNs, fileName) = try await ble.startRemoteRecording(clock: clock)
        self.bleAckMonotonicNs = ackNs
        self.cameraFileName = fileName
        #endif
    }

    public func stopRecording() async throws -> StreamStopReport {
        #if canImport(INSCameraSDK)
        try await ble.stopRemoteRecording()
        #endif
        return StreamStopReport(streamId: streamId, frameCount: 0, kind: "video")
    }

    public func ingest(into episodeDirectory: URL,
                       progress: @Sendable (Double) -> Void) async throws -> StreamIngestReport {
        #if canImport(INSCameraSDK)
        guard let fileName = cameraFileName else {
            throw Insta360Error.noRecordingToDownload
        }
        let dest = episodeDirectory.appendingPathComponent("\(streamId).mp4")
        let frameCount = try await wifi.download(remoteFileName: fileName, to: dest,
                                                  progress: progress)
        // Persist the BLE-ACK anchor alongside the video so the server knows
        // where to place the camera's internal clock on the host timeline.
        try writeInsta360SidecarAnchor(
            episodeDirectory: episodeDirectory,
            streamId: streamId,
            bleAckMonotonicNs: bleAckMonotonicNs)
        return StreamIngestReport(streamId: streamId,
                                  filePath: "\(streamId).mp4",
                                  frameCount: frameCount)
        #else
        throw Insta360Error.frameworkUnavailable
        #endif
    }

    public func disconnect() async throws {
        #if canImport(INSCameraSDK)
        try? await ble.unpair()
        #endif
        await healthBus?.publish(.streamDisconnected(streamId: streamId, reason: "normal"))
    }
}

public enum Insta360Error: Error {
    case frameworkUnavailable
    case noRecordingToDownload
    case bleNotPaired
    case wifiConnectionFailed(String)
    case downloadFailed(String)
    case commandFailed(String)
}

private func writeInsta360SidecarAnchor(episodeDirectory: URL,
                                        streamId: String,
                                        bleAckMonotonicNs: UInt64) throws {
    let obj: [String: Any] = [
        "stream_id": streamId,
        "ble_ack_monotonic_ns": bleAckMonotonicNs,
        "anchor_source": "ble_ack",
    ]
    let data = try JSONSerialization.data(withJSONObject: obj,
                                           options: [.prettyPrinted, .sortedKeys])
    try data.write(to: episodeDirectory
        .appendingPathComponent("\(streamId).anchor.json"), options: [.atomic])
}
```

Port egonaut's `Insta360CameraManager.swift` into this file. Key behaviours:

- `startRemoteRecording(clock:) async throws -> (ackMonotonicNs, fileName)`: send BLE "record start" command; wait for ACK; capture host `clock.nowMonotonicNs()` at ACK arrival; return the filename the camera reports
- `stopRemoteRecording()`: BLE "record stop" command
- `unpair()`: disconnect
- Publishes `HealthEvent.streamDisconnected` on unexpected drop

All INSCameraSDK symbol references MUST be inside `#if canImport(INSCameraSDK)` so the file compiles without the framework.

The implementation is ~250 lines of BLE+INSCameraSDK glue. Follow egonaut's source 1:1 — the SDK contract is brittle and tested there.
